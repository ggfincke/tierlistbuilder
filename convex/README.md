# convex/

Convex backend for tierlistbuilder. Schema, validators, auth, & function namespaces split by product area.

## First-time setup

1. **Install deps & start convex dev**

   ```sh
   npm install
   npx convex dev
   ```

   On first run, this walks you through creating a deployment & writes `VITE_CONVEX_URL` into `.env.local`. Leave `npx convex dev` running — it watches `convex/` for changes & regenerates `convex/_generated/`.

2. **Set the convex-side env vars**

   These live on the convex deployment, not in `.env.local`. Set them via the convex CLI:

   ```sh
   # the URL of your convex deployment's HTTP actions endpoint —
   # used as the JWT issuer for the @convex-dev/auth library
   npx convex env set CONVEX_SITE_URL https://<your-deployment>.convex.site

   # the URL of your frontend — used as the default redirect destination
   # after sign-in. http://localhost:5173 for `npm run dev`
   npx convex env set SITE_URL http://localhost:5173
   ```

   No OAuth client IDs or secrets are needed today — the only registered auth provider is `Password` (email + password). Google & Apple OAuth land in a follow-up PR & will add their own env vars then.

3. **Start the frontend & test sign-in**

   ```sh
   npm run dev
   ```

   Open the app, click the Settings button (top-right), scroll to the Account section, click **Sign in** → switch to the **Create account** tab, enter an email & a password (≥8 chars), submit. You should see your account in the AccountSection w/ a sign-out button. Sign out, then sign back in via the **Sign in** tab w/ the same credentials.

## Layout

```
convex/
  _generated/         # autogenerated by `npx convex dev`; gitignored
  lib/
    auth.ts           # getCurrentUser, requireCurrentUser
    hexColor.ts       # runtime hex-color validator for handler args
    permissions.ts    # requireBoardOwnership, requireTierPresetOwnership
    rateLimiter.ts    # shared rate-limit bucket defs & throw helper
    userUpsert.ts     # populates app-owned user fields on first sign-in
    validators.ts     # v.object() shapes mirroring packages/contracts
  platform/
    media/            # mediaAssets uploads, queries, & GC
    shortLinks/       # share-link slug resolution & TTL sweeper
  workspace/
    boards/           # boards CRUD + cascade delete
    settings/         # userSettings get/upsert
    sync/             # local/cloud board merge & reconciliation
    tierPresets/      # user preset CRUD
  auth.config.ts      # auth.js provider domain config
  auth.ts             # convexAuth() wiring + afterUserCreatedOrUpdated
  http.ts             # /auth/* HTTP routes
  schema.ts           # app tables + @convex-dev/auth authTables
  users.ts            # getMe query
```

## Adding a new function

1. Drop a `query`/`mutation`/`action` into the right namespace.
2. `npx convex dev` (running in another terminal) will regenerate `_generated/api.d.ts`.
3. Frontend imports the typed reference via `@convex/_generated/api`, e.g. `useQuery(api.workspace.boards.queries.getMyBoards, {})`.
4. UI components must NOT call convex queries directly — they go through `src/features/*/data/cloud/*Repository.ts` adapters or `src/features/platform/auth/model/useAuthSession`.

## Schema versioning

Don't bump `schema.ts` validators in a way that breaks existing rows w/o a migration. Convex doesn't auto-migrate — see https://docs.convex.dev/database/schemas#schema-validation for the migration playbook.
